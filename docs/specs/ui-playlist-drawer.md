# UI 詳細設計 — Playlist ドロワー

## 1. 概要

≡ ボタンをクリックすると右からスライドインするガラスドロワー。  
プレイリストの閲覧・追加・並び替え・削除・再生モード切り替えを提供する。

---

## 2. ドロワー本体仕様

```
幅:     320px (固定)
高さ:    ウィンドウ高さ 100%
位置:    右端 (x = ウィンドウ幅 - 320px)
背景:    rgba(10, 10, 15, 0.88)
        + backdrop-blur 近似
ボーダー左: 1px rgba(255, 255, 255, 0.12)

アニメーション:
  開く:  x: +320px → 0, duration 250ms, ease-out
  閉じる: x: 0 → +320px, duration 200ms, ease-in

Playlist ドロワーと Settings ドロワーが重なる場合:
  後から開いた方が最前面 (z-index 上位)
```

## 8. ドラッグ並び替え

### 8.1 操作フロー

```
1. ユーザーがドラッグハンドル (⠿) をマウスダウン
2. カーソルが grab → grabbing に変化
3. ドラッグ開始: アイテムが元の位置から浮き上がる
   (shadow: 0 8px 24px rgba(0,0,0,0.5), opacity: 0.95)
4. ドラッグ中: アイテムがカーソルに追従
5. 挿入位置ライン: ドロップ先に 2px の accent-purple ラインを表示
6. ドロップ: 新しい位置に配置, スムーズにスライドして定位置へ

アニメーション:
  ドロップ後の整列: 300ms ease-out
```

### 8.2 挿入位置ライン仕様

```
高さ: 2px
色:   accent-purple (#7c3aed)
幅:   ドロワー幅 - 左右パディング
位置: 挿入先のアイテムの上端 (hover 判定で最も近いギャップ)
アニメーション: フェードイン 100ms
```

### 8.3 Slint 実装注意

Slint の標準コンポーネントはドラッグ並び替えを直接サポートしないため、  
`TouchArea` と `Flickable` を組み合わせてカスタム実装する。

```slint
component DraggableListItem {
    in property <bool> is-dragging;
    in property <float> drag-y;  // ドラッグ中の Y オフセット

    y: is-dragging ? drag-y : 0;
    opacity: is-dragging ? 0.95 : 1.0;

    drop-shadow-blur:   is-dragging ? 24px : 0;
    drop-shadow-offset-y: is-dragging ? 8px : 0;
    drop-shadow-color:  rgba(0, 0, 0, 0.5);
}
```

## 9. コンテキストメニュー

### 9.1 表示トリガー

- **マウス右クリック**: デスクトップ環境で右クリック
- **長押し (500ms)**: タッチ環境 / Trackpad 向け

### 9.2 メニュー内容

```
┌─────────────────────────────────┐
│ ▶ 再生                           │
│ ─────────────────────────────── │
│ 🗑 リストから削除                 │
│ 📂 ファイルの場所を開く           │
│ ℹ️ メタデータを表示               │
└─────────────────────────────────┘
```

### 9.3 メニュースタイル

```
背景: rgba(15, 15, 20, 0.95)
ボーダー: 1px rgba(255,255,255,0.15)
角丸: 8px
shadow: 0 8px 32px rgba(0,0,0,0.7)
幅: 200px

アイテム:
  高さ: 36px, パディング 0 16px
  フォント: 13px
  ホバー: rgba(255,255,255,0.08)

区切り線: 1px rgba(255,255,255,0.10)

表示位置:
  右クリック位置を基準 (ウィンドウ端でクリップ)

閉じる条件:
  - メニュー外クリック
  - Esc キー
  - アイテム選択後 (150ms ディレイ)
```

### 9.4 各アクション

| アクション        | 動作                                           |
|---------------|------------------------------------------------|
| 再生           | `Command::LoadAndPlay(path)` — 即時再生開始      |
| リストから削除  | リストから除外 (ファイルは削除しない)               |
| ファイルの場所を開く | `open::that(parent_dir)` (open クレート使用) |
| メタデータを表示 | モーダルダイアログ: ビットレート/サンプルレート/フォーマット/長さ/ファイルサイズ |

## 13. Rust 側のデータモデル

```rust
// app/src/app/event.rs に追加
pub enum Event {
    // 既存 ...
    PlaylistUpdated(Vec<TrackInfo>),
    PlaylistModeChanged { shuffle: bool, repeat: RepeatMode },
}

pub enum RepeatMode { None, All, One }

// app/src/app/command.rs に追加
pub enum Command {
    // 既存 ...
    ReorderPlaylist { from: usize, to: usize },
    RemoveFromPlaylist(usize),
    ToggleShuffle,
    CycleRepeatMode,
    ClearPlaylist,
    OpenFileLocation(PathBuf),
}
```

## 14. パフォーマンス要件

| 項目                       | 目標値             |
|--------------------------|------------------|
| ドロワー開閉アニメーション     | 250ms / 200ms    |
| ドラッグ並び替えの追従        | ≤ 16ms (1 フレーム) |
| 1000 曲のリスト描画          | 仮想スクロール適用 (画面外のアイテムは非描画) |
| コンテキストメニュー表示       | ≤ 50ms           |
