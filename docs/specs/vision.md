# Sonic Flow - プロジェクトビジョン

## コンセプト

**音圧ビジュアライザー重視のハイクオリティ音楽プレイヤー**

画像素材に頼らず、FFT解析データとGPUシェーダーだけで美しいビジュアライザーを描画する。
音圧を精密に読み取り、波のようにリアルタイムでアニメーションさせることが最重要目標。

## 技術スタック

- **言語**: Rust
- **UI**: Slint (再生コントロール、プレイリスト、設定画面)
- **GPU描画**: wgpu (ビジュアライザー、シェーダーエフェクト)
- **オーディオ**: rodio / cpal (再生・音声取得)
- **FFT**: rustfft (周波数解析)
- **非同期**: tokio

## アーキテクチャ概要

```
音楽ファイル
  |
  v
オーディオ再生 (rodio/cpal)
  |
  v
FFT解析 (rustfft)
  |
  v
周波数データ (低音/中音/高音)
  |
  v
GPU転送 (uniform buffer)
  |
  v
シェーダー描画 (wgpu + WGSL)
```

### Slint 担当

- 再生コントロール (再生/停止/スキップ)
- トラック情報表示
- プレイリスト管理
- ボリュームスライダー
- 設定画面
- テーマ (ダーク/ライト)

### wgpu 担当

- ビジュアライザー描画 (全てシェーダーで)
- パーティクル
- ポストプロセスエフェクト (グロー、ブルーム、グリッチ等)

## ビジュアライザー種別

全てシェーダーのみで描画する。画像素材は使わない。

### 1. スペクトラムバー

FFT結果から周波数ごとの強さを棒グラフ状に描画。
ピークホールド、グラデーション、グロー付き。

### 2. 波形 (オシロスコープ)

波形データをそのまま描画。ラインにグロー/ブルームを適用。

### 3. 円形ビジュアライザー

音圧に応じてリング形状が変形。
周波数帯域ごとに異なる変形を適用。

### 4. パーティクル

音圧に応じて発光・拡散・回転・ノイズ変形。
プロシージャル生成 (テクスチャ不要)。

### 5. 背景エフェクト

- グロー / ブルーム
- パーリンノイズ
- 波紋
- グリッチ
- クロマティックアバレーション

全てフラグメントシェーダーで実装。

## FFT解析の活用

音圧を精密に読み取り、周波数帯域ごとに分離してビジュアルに反映する。

```
低音 (20-250Hz)   -> 大きな動き、カメラ揺れ、背景の脈動
中音 (250-4kHz)   -> 波形変形、メインビジュアルの変化
高音 (4kHz-20kHz) -> パーティクル、きらめき、ディテール
```

## パフォーマンス目標

| 項目 | 目標値 |
|------|--------|
| ビジュアライザー描画 | 120fps (8.3ms以下) |
| オーディオ遅延 | 50ms以下 |
| UI応答性 | 60fps (16ms以下) |
| メモリ使用量 | 100MB以下 (idle) / 200MB以下 (active) |
| CPU使用率 | 5%以下 (再生時) |
| GPU使用率 | 30%以下 |

## 素材が必要になるケース

ビジュアライザー自体はシェーダーのみだが、以下には素材が必要:

- アプリアイコン / ロゴ
- UIアイコン (再生、停止、スキップ等)
- アルバムアート (メタデータから取得)
