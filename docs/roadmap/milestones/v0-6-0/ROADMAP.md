# v0.6.0 - ライブラリ管理

## 概要

複数の音楽ファイルを管理するライブラリ機能を実装する。フォルダスキャン、永続化、検索/フィルタを備え、日常使い可能なプレイヤーへ進化させる。

## ゴール

- フォルダ追加 → 全音楽ファイル自動検出
- ライブラリの永続化 (再起動後も保持)
- 検索 / ソート / フィルタ
- 1万曲規模で動作

## スコープ

### In Scope

- ライブラリスキャン (再帰的フォルダ走査)
- メタデータキャッシュ
- 永続化 (SQLite または JSON)
- ライブラリビュー UI
- ファイルシステム監視 (追加/削除を検知)

### Out of Scope

- プレイリスト機能 (v0.7.0)
- スマートプレイリスト (v1.x)
- ネットワーク経由のメディア (v1.x 以降)
- アルバムアートのキャッシュ生成 (v0.8.0 以降)

## リスク / 技術的検討事項

- **SQLite vs JSON**: SQLite は依存が大きいが検索性能が圧倒的。1万曲スケールなら SQLite が現実解
- **長時間スキャン**: 大量ファイルではメタデータ抽出が遅い。段階的処理 (パス → 後でメタデータ) で UX 改善
- **メタデータの欠如**: 全フォーマットでメタデータ抽出が成功するとは限らない (壊れたタグ等)。フォールバック (ファイル名から推測) を用意
- **パスの扱い**: ネットワークドライブ/移動可能ストレージ上のファイル管理に注意
