# v0.5.0 - ポストプロセス & エフェクト

## 概要

ブルーム、グロー、グリッチ等のポストプロセスエフェクトを導入し、ビジュアライザーの「魅力」を引き上げる。マルチパスレンダリングパイプラインと HDR 対応を整備する。

## ゴール

- ブルーム / グロー / クロマティックアバレーション / フィルムグレイン等の標準エフェクト
- ビジュアライザーごとにエフェクトプリセット
- HDR レンダリング (f16 テクスチャ)
- フレームレート目標を維持 (全エフェクト有効でも 60fps)

## スコープ

### In Scope

- マルチパスレンダリングパイプライン (offscreen render targets)
- 標準ポストエフェクトの実装
- HDR + トーンマッピング
- エフェクト UI 制御

### Out of Scope

- パーティクルシステム (将来検討、v1.x)
- 3D 効果 (v1.x 以降)
- カスタムエフェクト言語 (プラグイン以降)

## タスク

### マルチパスパイプライン

- [ ] Offscreen render target の管理 (RT0 / RT1 / ping-pong)
- [ ] Pass 抽象 (`RenderPass` trait)
- [ ] Pass チェイン (Visualizer Pass → Bloom Pass → ... → Final Composite)
- [ ] テクスチャフォーマット選択 (sRGB / linear / HDR)
- [ ] 解像度スケーリング (フルHDR → 半分解像度ブルームなど)

### 標準エフェクト実装

- [ ] **Gaussian Blur** (separable horizontal/vertical, 5x5 / 9x9)
- [ ] **Bloom** (luma threshold + downsample + blur + upsample + add)
- [ ] **Glow** (シーン全体への発光増幅)
- [ ] **Chromatic Aberration** (RGB 各チャネルのオフセット)
- [ ] **Film Grain** (ノイズオーバーレイ, 強度可変)
- [ ] **Vignette** (エッジ暗減衰)
- [ ] **Color Grading** (LUT or 簡易 contrast/saturation)

### HDR レンダリング

- [ ] f16 テクスチャ対応の確認 (デバイス feature クエリ)
- [ ] sRGB / Linear 空間の管理
- [ ] トーンマッピング (Reinhard, ACES Filmic)
- [ ] 非対応デバイスへのフォールバック (LDR モード)

### エフェクトスタック

- [ ] エフェクトの ON/OFF 切替
- [ ] 各エフェクトのパラメータ調整
- [ ] プリセット (ビジュアライザーごとの推奨設定)
- [ ] プリセット保存 (`Default`, `Subtle`, `Intense`, `Cinematic`)

### UI 統合

- [ ] エフェクトトグル UI (Slint コンポーネント)
- [ ] Intensity スライダー
- [ ] プリセット選択

### パフォーマンス検証

- [ ] 全エフェクト有効でも 60fps
- [ ] フレームタイム分散 < 2ms
- [ ] GPU メモリ使用量 < 50MB
- [ ] 解像度ダウンサンプルでのコスト低減確認

## 完了条件

- [ ] Bloom が見える (明るいバーが光る)
- [ ] エフェクトの ON/OFF 切替が即座に反映
- [ ] フレームレート維持 (60fps with all effects)
- [ ] HDR ディスプレイで滑らかな明部表現
- [ ] プリセットでサクッと雰囲気変更

## 依存

- v0.4.0 (複数ビジュアライザーが揃っていることで、エフェクトの効きを比較できる)

## リスク / 技術的検討事項

- **GPU メモリ管理**: ping-pong バッファでメモリ消費が倍増するため、解像度スケーリングが必須
- **HDR 互換性**: 古いデバイスは f16 テクスチャレンダリング非対応。検出してフォールバック必須
- **トーンマッピング選択**: ACES は印象的だがプロジェクトの色味方向性次第。デザイン方針を決める必要
